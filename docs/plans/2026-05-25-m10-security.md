# m10 — `crates/security` (crypto + connection policy) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use `lore:execute` to implement this plan task-by-task.
> **Scope guard:** Implements milestone m10 of `docs/plans/2026-05-25-engine-phase-roadmap-m9-m15.md`. Do ONLY `crates/security`. Note adjacent work for other crates as TODO and continue.

**Goal:** Make `crates/security` a working library providing real, correct cryptography and connection policy: Argon2id password hashing, ChaCha20-Poly1305 secret encryption, an in-memory credential store, a governor-backed firewall (rate limit + IP allow/block), and a security audit log — all tested and clippy/fmt-clean.

**Architecture:** Four modules behind small typed contracts: `encryption` (`PasswordHasher`, `SecretCipher`), `credential` (`CredentialStore`), `firewall` (`Firewall`), `audit` (`AuditLog`). Crypto correctness is the priority — no fake/placeholder hashing or encryption ever.

**Tech Stack:** Rust edition 2024. `argon2` (Argon2id, PHC strings) · `chacha20poly1305` (RustCrypto AEAD) · `governor` (rate limiting) · `thiserror`. Decisions locked: **ChaCha20-Poly1305**, **Argon2id**, **governor**.

**Practices:** TDD (red→green) · typed-interfaces-first + contract-first · `cargo clippy --workspace --all-targets -- -D warnings` + `cargo fmt --check` gate per task · `thiserror` typed errors. **No stubs — real crypto, real round-trips in tests.**

**Required skills:** none.

## External-API reality (READ FIRST — technique from m6/m9, 3-for-3)

`argon2`, the RustCrypto AEAD crates, and `governor` change APIs across minor versions. Code blocks here are best-effort against the pinned versions. In every task that touches these crates, **verify the real signatures against the resolved source first**:

```bash
find ~/.cargo/registry/src/index.crates.io-* -maxdepth 1 -type d \
  \( -name 'argon2-*' -o -name 'chacha20poly1305-*' -o -name 'governor-*' -o -name 'password-hash-*' -o -name 'aead-*' \)
```
Read the resolved versions for: `Argon2::default()`, `PasswordHasher::hash_password`, `PasswordHash::new` + `PasswordVerifier::verify_password`, `SaltString::generate`, the `OsRng` re-export path (`argon2::password_hash::rand_core::OsRng`), `ChaCha20Poly1305::new`/`generate_nonce`/`encrypt`/`decrypt` + the `aead::OsRng` path, and governor's `RateLimiter::keyed`, `Quota`, `clock::{DefaultClock, FakeRelativeClock}`. **Never weaken crypto or stub a path to compile — fix to the real API.**

**Pinned cohort (root `[workspace.dependencies]`):**

Confirm resolved versions with `cargo tree` in T0; if a materially newer minor resolves, pin the exact version matching the code or adapt.

## Task 1: `PasswordHasher` — Argon2id — TDD

**Files:** `encryption/hash.rs`, `encryption/mod.rs`.

**Step 1 — verify** the argon2 0.5 API + `OsRng` path (see preamble).

**Step 2 — failing tests first** (`hash.rs`):
```rust
#[cfg(test)]
mod tests {
    use super::*;
    #[test]
    fn hash_then_verify_roundtrips() {
        let h = PasswordHasher::default();
        let phc = h.hash("correct horse").unwrap();
        assert!(h.verify("correct horse", &phc).unwrap());
    }
    #[test]
    fn wrong_password_fails_verification() {
        let h = PasswordHasher::default();
        let phc = h.hash("correct horse").unwrap();
        assert!(!h.verify("Tr0ub4dor", &phc).unwrap());
    }
    #[test]
    fn same_password_hashes_differ_by_salt() {
        let h = PasswordHasher::default();
        assert_ne!(h.hash("pw").unwrap(), h.hash("pw").unwrap());
    }
}
```

**Step 3 — run:** `cargo test -p security` → FAIL.

**Step 4 — implement** a real `PasswordHasher` wrapping `argon2::Argon2` (Argon2id, default params). `hash` generates a random `SaltString` via the re-exported `OsRng`, returns the PHC string; `verify` parses the PHC string with `PasswordHash::new` and checks via `PasswordVerifier::verify_password`. Map argon2 errors → `SecurityError::Hash`. The salt is generated inline with `SaltString::generate(&mut OsRng)` in `hash.rs`. No hardcoded salts; no fake hashing.

**Step 5:** `cargo test -p security` → PASS. **Step 6 — gate + commit** (`feat(security): Argon2id PasswordHasher`).

## Task 2: `SecretCipher` — ChaCha20-Poly1305 — TDD

**Files:** `encryption/encrypt.rs`, `encryption/decrypt.rs`, `encryption/manager.rs`, `encryption/mod.rs`.

**Step 1 — verify** the `chacha20poly1305` 0.10 API (`KeyInit::new`, `generate_nonce`, `aead::{Aead, OsRng}`).

**Step 2 — failing tests first** (`encrypt.rs` or `manager.rs`):
```rust
#[cfg(test)]
mod tests {
    use super::*;
    #[test]
    fn encrypt_then_decrypt_roundtrips() {
        let c = SecretCipher::generate();
        let sealed = c.encrypt(b"oauth-token").unwrap();
        assert_eq!(c.decrypt(&sealed).unwrap(), b"oauth-token");
    }
    #[test]
    fn tampered_ciphertext_fails() {
        let c = SecretCipher::generate();
        let mut sealed = c.encrypt(b"secret").unwrap();
        *sealed.last_mut().unwrap() ^= 0xFF;
        assert!(c.decrypt(&sealed).is_err());
    }
    #[test]
    fn wrong_key_fails() {
        let sealed = SecretCipher::generate().encrypt(b"secret").unwrap();
        assert!(SecretCipher::generate().decrypt(&sealed).is_err());
    }
}
```

**Step 3 — run:** FAIL. **Step 4 — implement** `SecretCipher` over `ChaCha20Poly1305`. `encrypt` generates a fresh random 96-bit nonce, returns `nonce || ciphertext` (prepended); `decrypt` splits the nonce and verifies the tag. `generate()` makes a random key; `from_key(&[u8;32])` builds one. Map AEAD errors → `SecurityError::{Encrypt,Decrypt}`. Nonces are never reused (a fresh random nonce per encryption, prepended to the ciphertext).

**Step 5:** PASS. **Step 6 — gate + commit** (`feat(security): ChaCha20-Poly1305 SecretCipher`).

## Task 3: `CredentialStore` — TDD

**Files:** `credential/provider.rs`, `credential/manager.rs`, `credential/mod.rs`.

**Ownership (locked):** the **store holds a `PasswordHasher`** and calls `hasher.verify(pw, &credential.phc)`; a `Credential` carries only data (username, PHC string, optional encrypted secret) — no verification logic on the credential itself.

**Step 1 — contract + failing tests.** Define `Credential` (a username + Argon2 PHC password hash + optional encrypted secret blob) and a `CredentialStore` trait (`put`, `get`, `verify_password`). TDD an in-memory impl: store a credential, verify the right password, reject the wrong one, retrieve an encrypted secret round-tripped through `SecretCipher`.

**Step 2 — run:** FAIL. **Step 3 — implement** the trait + an in-memory `MemoryCredentialStore` (a `HashMap<String, Credential>` behind a `RwLock`), composing T1's `PasswordHasher` and T2's `SecretCipher`. `provider.rs` = the trait + types; `manager.rs` = the in-memory impl. Persistent backing is **deferred** (the trait lets storage/m12 add a DB impl later).

**Step 4:** PASS. **Step 5 — gate + commit** (`feat(security): credential store over hasher + cipher`).

## Task 4: `Firewall` — governor rate limit + IP allow/block — TDD

**Files:** `firewall/{firewall.rs,config.rs,manager.rs,mod.rs}` (allow/block/track/trace/pause are fields/methods of the `Firewall` type, not separate files — see the skip note).

**Step 1 — verify** governor 0.6 API: `RateLimiter::keyed(Quota)`, `Quota::per_second(NonZeroU32)`, `.check_key(&key)`, and `clock::{DefaultClock, FakeRelativeClock}` for deterministic tests.

**Step 2 — failing tests first** using `FakeRelativeClock` (no real time):
```rust
#[cfg(test)]
mod tests {
    use super::*;
    use std::net::{IpAddr, Ipv4Addr};
    fn ip(n: u8) -> IpAddr { IpAddr::V4(Ipv4Addr::new(10, 0, 0, n)) }

    #[test]
    fn blocklisted_ip_is_denied_regardless_of_rate() { /* add to block set → check() == Deny */ }
    #[test]
    fn allowlisted_ip_bypasses_rate_limit() { /* allow set → always Allow */ }
    #[test]
    fn rate_limit_denies_after_quota_then_recovers_with_clock() {
        // FakeRelativeClock: exhaust quota → Deny; advance clock → Allow again
    }
}
```

**Step 3 — run:** FAIL. **Step 4 — implement** the `Firewall` (`firewall.rs`): it holds an IP allow set, an IP block set, a per-key governor `RateLimiter` over an injected clock, a bounded recent-`Deny` ring (for observability), and an enforcement-pause flag. `check(ip) -> Decision { Allow, Deny(reason) }` consults block → allow → rate-limit in that order, and short-circuits to `Allow` when paused. `config.rs` = `FirewallConfig` (quota + allow/block seed sets); `manager.rs` = building a `Firewall` from config. The clock is injected so tests drive it with `FakeRelativeClock`.

**Step 5:** PASS. **Step 6 — gate + commit** (`feat(security): governor-backed firewall with allow/block`).
